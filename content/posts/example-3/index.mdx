# Interactive Graphics
In the previous tutorial, we tried input element slider, however, graphics primitives can also act like an virtual input device.

There are several events we can capture, but lets focus on `zoom` and `drag`

<wljs-editor display="codemirror" type="Input"  >{String.raw`udata = {{-1,0}, {1,0}};

Graphics[{
  Cyan, Line[udata // Offload], Red, PointSize[0.05],
  EventHandler[Point[{0,0.5}], {
    "drag" -> drag,
    "zoom" -> zoom
  }]
}, PlotRange->{{-1,1}, {0,1}}]`}</wljs-editor>

Ok, then lets generate the data

<wljs-editor display="codemirror" type="Input"  >{String.raw`ClearAll[generate];
generate[{x_, y_}, k_] := Table[{t, y Exp[- (*FB[*)(((*SpB[*)Power[(x - t)(*|*),(*|*)2](*]SpB*))(*,*)/(*,*)(2 k))(*]FB*)]}, {t,-1,1, 0.03}]`}</wljs-editor>

We should generate it everytime, when we have interact with a dot

<wljs-editor display="codemirror" type="Input"  >{String.raw`defaults = {{0, 0.5}, 1/20.};

drag[xy_] := (
  defaults[[1]] = xy;
  udata = generate @@ defaults;
)

zoom[k_] := (
  defaults[[2]] = k / 20.;
  udata = generate @@ defaults;
)`}</wljs-editor>
